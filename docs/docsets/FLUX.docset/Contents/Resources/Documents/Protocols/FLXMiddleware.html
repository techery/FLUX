FLXMiddleware

@protocol FLXMiddleware <NSObject>

Provides an interface for generating of side-effects to some FLUX events. Each middleware instance is notified about all events and each middleware is responsible for filtering only events it is interested in.

All middleware methods are called in a context provided by domain’s executor. That’s why it is considered as a good practice to apply side-effect asynchronously in order to prevent blocking action processing by stores.
