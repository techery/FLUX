FLXDomain

@interface FLXDomain : NSObject <FLXDomainProtocol>

Main FLUX class that is designed to be subclassed by your application domain.

Warning

This class is not designed for subclassing. Basically the only reason for subclassing is to extend base implementation with more convenience accessors for stores, e.g. wrap @link -(void)storeByClass:(Class)storeClass @/link to @link -(CustomStoreClass)mySpecificStore @/link method.
